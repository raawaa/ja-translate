# 为双语对照翻译添加样式区分

## 问题分析
项目当前已经实现了双语对照的HTML结构，通过添加CSS类名区分原文和译文：
- `bilingual-container`：包含原文和译文的容器
- `original-text`：原文段落
- `translated-text`：译文段落

但是，目前没有为这些类名定义对应的CSS样式，导致原文和译文在视觉上没有区分。

## 解决方案
1. **创建CSS文件**：创建一个单独的CSS文件，保留三个CSS类名，但仅定义原文的字体颜色，淡化原文
2. **修改翻译脚本**：确保CSS文件被复制到输出目录
3. **更新HTML文件**：确保CSS文件被正确引入到HTML文件中

## 实现步骤

### 1. 创建CSS文件
创建 `source/OEBPS/css/bilingual.css` 文件，添加以下样式：
```css
/* 双语对照样式 */
/* 容器样式（保留类名，无样式） */
.bilingual-container {
    /* 容器样式可在后续扩展 */
}

/* 原文样式 - 淡化处理 */
.original-text {
    color: #666;
}

/* 译文样式（保留类名，无样式） */
.translated-text {
    /* 译文样式可在后续扩展 */
}
```

### 2. 修改epub_translator.py
确保CSS文件被复制到输出目录，并在HTML文件中添加CSS引用。

### 3. 更新HTML文件处理逻辑
修改 `update_file_content_by_type_incremental` 函数，确保每个HTML文件都引入了CSS文件。

## 预期效果
- 原文：灰色字体，视觉上被淡化
- 译文：保持默认样式，正常显示

## 文件修改列表
1. `source/OEBPS/css/bilingual.css`：创建新的CSS文件
2. `epub_translator.py`：修改HTML处理逻辑，添加CSS引用

## 注意事项
- 确保CSS文件路径与项目的实际目录结构一致
- 考虑到不同 EPUB 阅读器的兼容性，使用基础的CSS属性
- 在CSS文件中保留三个类名，但仅对原文样式设置颜色
