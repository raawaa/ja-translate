## 实现原文淡化效果

### 问题分析
目前双语对照功能已经实现，但没有添加CSS样式，导致原文和译文看起来没有区别。用户希望原文淡化，让读者更容易阅读翻译后的文本。

### 实现方案
1. **创建单独的CSS文件**：新建一个CSS文件来定义双语对照的样式
2. **在HTML中引用CSS文件**：处理HTML文件时，添加对CSS文件的引用
3. **设置原文淡化样式**：在CSS文件中为`original-text`类添加淡化效果
4. **确保CSS文件正确打包**：确保CSS文件被包含在最终的EPUB文件中

### 实现步骤
1. **新建CSS文件**：创建`bilingual.css`文件，定义双语对照样式
2. **修改HTML处理逻辑**：在处理HTML文件时，检查并添加CSS文件引用
3. **确保引用唯一性**：检查HTML是否已引用CSS文件，避免重复添加
4. **设置原文淡化样式**：在CSS中定义`original-text`类的淡化效果
5. **确保CSS文件打包**：检查EPUB打包逻辑，确保CSS文件被正确包含

### 预期效果
- 原文显示为淡化效果（透明度0.6或浅灰色）
- 译文正常显示，清晰可读
- 双语对照结构保持完整
- 样式集中管理，便于维护和更新

### 关键代码修改
1. **创建`bilingual.css`文件**：
   ```css
   .original-text {
       opacity: 0.6;
       color: #666;
       margin-bottom: 0.5em;
   }
   
   .translated-text {
       font-weight: normal;
   }
   
   .bilingual-container {
       margin-bottom: 1em;
   }
   ```

2. **修改`update_file_content_by_type_incremental`函数**：添加CSS文件引用逻辑
3. **检查EPUB打包逻辑**：确保CSS文件被正确包含在EPUB中

### 实现细节
1. **CSS文件位置**：将CSS文件放在EPUB的样式目录中
2. **引用路径**：根据EPUB的目录结构设置正确的CSS引用路径
3. **引用检查**：通过检查`link`标签判断CSS是否已引用
4. **兼容性处理**：支持不同的HTML结构
5. **性能优化**：只在需要时添加引用，避免不必要的处理

这个实现将使双语对照功能更加实用和易于维护，让读者能够更轻松地阅读翻译后的文本，同时保留原文作为参考。
